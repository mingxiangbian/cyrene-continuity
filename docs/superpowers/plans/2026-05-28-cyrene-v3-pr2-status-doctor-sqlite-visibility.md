# Cyrene v3 PR2 Status / Doctor / SQLite Visibility Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让用户能通过 `cyrene-continuity codex memory status` 一条命令判断 memory pipeline 是否健康，并让 `doctor/status` 清楚暴露 Node / SQLite / fallback / stale index / Stop Hook / projectId 状态。

**Architecture:** 新增一个只读 status snapshot 层，集中读取 memory roots、counts、SQLite diagnostics、index freshness、Stop Hook、project identity，再分别给 CLI status 和 doctor 复用。status 只读文件和 adapter diagnostics，不调用 `rebuildFromRoots()`、`syncRoot()` 或任何会创建/写入 index 的 API。

**Tech Stack:** TypeScript, Vitest, Node fs promises, existing Codex CLI, existing memory JSONL store, existing SQLite adapter diagnostics.

- [ ] **Step 3: Implement read-only snapshot and formatter**

Create `src/codex/codex-memory-status.ts` with these exported entry points:

```ts
export async function readCodexMemoryStatus(input: { cwd: string }): Promise<CodexMemoryStatus>
export async function formatCodexMemoryStatus(input: { cwd: string }): Promise<string>
```

Implementation requirements:

- Use `identifyCodexProject(input.cwd)` for `projectId`, `displayName`, `gitRoot`, `gitRemoteHash`.
- Use `codexGlobalMemoryRoot()` / `codexProjectMemoryRoot(projectId)` for root paths.
- Use `readActiveMemoriesFromRoot()`, `readPendingMemoriesFromRoot()`, and `readTombstonesFromRoot()` for counts.
- Use `isCodexStopHookConfigured()` for Stop Hook status.
- Use `readCodexMemoryIndexDiagnostics()` for SQLite availability and reason.
- Determine fallback mode as `sqlite` when `diagnostics.available === true`, otherwise `jsonl`.
- Determine index freshness by comparing `memory.db` mtime against latest `index.jsonl` / `pending.jsonl` mtime in readable global/project roots:
  - `unavailable` when SQLite diagnostics are unavailable.
  - `stale` when a source JSONL exists but `memory.db` is missing or older.
  - `fresh` when `memory.db` exists and is newer than indexed source JSONL.
  - `empty` when no indexed source JSONL exists.
- Do not call `rebuildCodexMemoryIndex()`, `syncCurrentCodexMemoryIndex()`, adapter `initialize()`, `rebuildFromRoots()`, or `syncRoot()`.

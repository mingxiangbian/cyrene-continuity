# Cyrene v3 PR6 Index Retrieval Stabilization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 完成 v3 PR6：让 `cyrene_continuity_get` 成为真正只读的 retrieval path；当 SQLite index stale 或 unavailable 时返回可解释 diagnostics，并安全 fallback 到 JSONL，不把 pending memory 当 active fact。

**Architecture:** 抽出一个只读 `codex-memory-index-status` helper，统一给 status/doctor 和 continuity retrieval 读取 SQLite availability、freshness、fallback mode。`continuity-context` 只在 freshness 为 `fresh` 时 query SQLite；任何 stale/unavailable/empty index 都走 JSONL fallback，并把 `source`、`routes`、`freshness`、`fallbackMode`、`staleReason` 回传到 `diagnostics.memoryIndex`。显式 rebuild 仍只通过 `cyrene-continuity codex memory db rebuild` 执行。

**Tech Stack:** TypeScript, Node fs/promises, Vitest, existing memory JSONL store, existing SQLite adapter diagnostics, MCP stdio smoke tests.

**Files:**

- [ ] **Step 1: Extract read-only status helper**

Create `src/codex/codex-memory-index-status.ts` with these exports:

```ts
export type CodexMemoryFallbackMode = 'sqlite' | 'jsonl'
export type CodexMemoryIndexFreshness = 'fresh' | 'stale' | 'empty' | 'unavailable'

export interface CodexMemoryIndexStatus {
  available: boolean
  dbPath: string
  ftsTokenizer?: MemoryIndexDiagnostics['ftsTokenizer']
  reason?: string
  fallbackMode: CodexMemoryFallbackMode
  freshness: CodexMemoryIndexFreshness
  lastSyncAt?: string
  sourceLatestAt?: string
  staleReason?: string
}

export async function readCodexMemoryIndexStatus(memoryRoots: string[]): Promise<CodexMemoryIndexStatus>
```

Implementation requirements:

- Call `readCodexMemoryIndexDiagnostics()` for adapter diagnostics only.
- Use `lstat()` to reject a directory/non-file `memory.db` path and force `available: false`.
- Compare `memory.db` mtime with latest readable `index.jsonl` / `pending.jsonl` mtime from `memoryRoots`.
- Return `freshness: 'unavailable'` when diagnostics are unavailable.
- Return `freshness: 'empty'` when no indexed source JSONL exists.
- Return `freshness: 'stale'` when source exists but DB is missing or older by more than 1000ms.
- Return `freshness: 'fresh'` only when DB exists and is not older than source.

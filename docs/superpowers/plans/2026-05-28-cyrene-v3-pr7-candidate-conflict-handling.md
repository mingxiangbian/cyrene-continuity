# Cyrene v3 PR7 Candidate Classification / Conflict Handling Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 完成 v3 PR7：让 pending memory 的候选分类成为可持久化、可 hash 审查的字段；在 approve/promote 时对相同 `normalizedKey` 的 active memory 做显式冲突门禁；支持 `supersede` / `keep_both` / `reject_new` 三种决议并写入可追踪事件；禁止 `open_question` 候选进入 active memory。

**Architecture:** 保持现有 pending-review 主流程和 JSONL store 不做大改。新增一个小型 candidate-kind helper，统一读取 `candidateKind` / legacy `candidate_kind` / tag-derived kind；review hash 纳入规范化后的 kind。`promoteCodexPendingMemory()` 在 hash 与 validator 通过后、写入前检测同 root active memories 中的 `normalizedKey` 冲突；没有显式 resolution 时返回不变更文件的 conflict result。resolution 由 core API 接收，CLI/MCP 只是透传并做 enum 校验。supersede 通过 active index 移除旧 memory、写 superseded tombstone 和 event 来追踪；keep_both 保留双方并写 audit event；reject_new 删除 pending、写 rejected tombstone 和 audit/reject event。

**Tech Stack:** TypeScript, Node fs/promises, Vitest, existing Codex memory review CLI, existing MCP stdio tools, JSONL memory store.

- `keep_both` keeps old active memory, promotes the new memory, and writes a conflict-resolution audit event.
- `reject_new` removes the pending candidate, leaves active memory unchanged, writes a rejected tombstone, and writes a conflict-resolution audit event.

- invalid CLI resolution is rejected with a clear error.
- MCP `handleMemoryPromote()` accepts `conflictResolution: 'keep_both'`.

- [ ] **Step 2: Implement `MemoryConflictResolution` input plumbing**

Add optional `conflictResolution` to:

- `runCodexMemoryApprove()` input.
- CLI parser via `parseOptionalConflictResolution()`.
- MCP promote handler and zod schema.

Use CLI spellings `supersede`, `keep-both`, `reject-new`; normalize to core spellings `supersede`, `keep_both`, `reject_new`.

- [ ] **Step 3: Implement resolution writes**

For `supersede`:

- remove conflicting active memory ids from active index.
- promote new memory with `supersedes` containing conflict ids.
- append one `superseded` tombstone per old memory with `replacementMemoryId`.
- append `action: 'supersede'` event with resolution details.

For `keep_both`:

- append new active memory without replacing same-key old memory.
- append `action: 'audit'` event with resolution details.

For `reject_new`:

- remove pending candidate.
- append rejected tombstone for candidate.
- append `action: 'reject'` plus `action: 'audit'` event, or one audit event that contains the rejection details if existing tests prefer a single event.
